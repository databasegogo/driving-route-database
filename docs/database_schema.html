一
核心地圖 & 事故資料表
district 行政區
儲存地圖中的行政區域資訊。本專案以龜山區為空間查詢範圍，所有道路與事故資料皆限定在此區域內。
屬性名稱 資料型別 鍵值 詳細說明
district_id INTEGER PK 行政區的唯一識別碼，對應 OSM 來源資料的 gid 欄位。
osm_id TEXT — OpenStreetMap 原始資料的物件 ID，保留此欄位以便日後追溯資料來源或重新匯入。
name TEXT — 行政區的中文名稱，例如「龜山區」。此欄位不可為 NULL。
admin_level TEXT — 行政層級，對應 OSM 的 fclass。本專案篩選 admin_level7，即台灣鄉鎮市區層級。
geom geometry — 行政區的地理邊界多邊形（Polygon），座標系統為 WGS84（EPSG:4326）。建有 GIST 空間索引，可高效執行「點是否在區內」等空間查詢。
road 道路
儲存龜山區內每一條道路的基本屬性，資料來源為 OpenStreetMap 路網資料集。
屬性名稱資料型別鍵值詳細說明
road_idINTEGERPK道路唯一識別碼，對應來源資料的 gid。
osm_idTEXT—OpenStreetMap 原始道路 ID，可回查 OSM 地圖確認道路真實資訊。
nameTEXT—道路名稱，例如「文化二路」。無名小路或私人道路此欄位可能為 NULL。
road_classTEXT—道路類型，對應 OSM 的 fclass，例如 residential（住宅區道路）、primary（主要幹道）、motorway（高速公路）、footway（人行道）等。可用於篩選適合車輛行駛的路段。
district_idINTEGERFK外鍵，關聯到 district 資料表，標示此道路所屬的行政區。不可為 NULL。
onewayTEXT—行駛方向限制。yes 表示單向行駛（正向）、no 表示可雙向通行、-1 表示僅能反向行駛（逆向單行）。此資訊會影響路段邊的 reverse_cost。
maxspeedSMALLINT—法定最高速限（公里 / 小時）。若 OSM 原始資料未記錄速限，此欄位為 NULL。可用於未來行駛時間估算功能。
bridgeTEXT—是否為橋梁結構。yes 表示是橋，路線規劃時可依使用者偏好（avoid_bridge）排除此類路段。
tunnelTEXT—是否為隧道結構。yes 表示是隧道，同樣可依 avoid_tunnel 偏好排除。
geomgeometry—道路的地理線條（LineString），表示道路實際走向與位置，座標系統為 WGS84，建有 GIST 空間索引。
road_edge 路段邊（圖論）
將道路拆成圖論中具有方向性的「邊（edge）」。每條邊有起終節點與通行成本，是 pgRouting 最短路徑演算法的核心輸入資料。
屬性名稱資料型別鍵值詳細說明
edge_idINTEGERPK路段邊的唯一識別碼。
road_idINTEGERFK外鍵，關聯到 road，代表此邊屬於哪條道路。一條道路可拆出多條邊（在交叉路口處被切割）。
sourceINTEGER—邊的起始節點 ID，代表一個道路交叉口或端點，由 pgRouting 的拓撲建構工具自動產生。
targetINTEGER—邊的終止節點 ID，與 source 共同定義邊的方向（source → target）。
costDOUBLE PRECISION—正向通行成本（從 source 到 target）。通常以距離（公尺）表示，-1 表示此方向不可通行（例如單行道反向）。
reverse_costDOUBLE PRECISION—反向通行成本（從 target 到 source）。若是單行道，此值設為 -1 代表反向不可行駛。雙向道路則與 cost 相同。
lengthDOUBLE PRECISION—路段的實際長度（公尺）。用於計算事故密度（edge_risk_score）及路線總距離統計。
geomgeometry—路段的地理線條，表示此段路的實際空間位置，建有 GIST 索引供空間查詢使用。
accident_severity 事故嚴重度代碼
參考代碼表（Lookup Table），定義事故嚴重程度的分類。參考台灣交通部標準，避免在事故主表中重複儲存文字描述。
屬性名稱資料型別鍵值詳細說明
severity_idSERIALPK嚴重度的系統自動流水號識別碼。
severity_codeTEXTUK事故類型代碼，不可重複。A1 = 死亡事故、A2 = 受傷事故、A3 = 財損事故，對應台灣警政署事故資料分類。
severity_nameTEXT—嚴重度的中文全名，例如「死亡事故」、「受傷事故」，用於前端介面顯示。
accident 事故記錄
儲存每筆交通事故的核心資料。整合 A1（死亡）、A2（受傷）兩種來源資料集，統一格式後存入。
屬性名稱資料型別鍵值詳細說明
accident_idSERIALPK事故在本資料庫中的系統自動流水號，作為內部唯一識別碼。
source_typeTEXT—事故來源資料集的類型，例如 A1 或 A2。搭配 source_accident_id 組成複合唯一鍵，用於避免重複匯入。
source_accident_idINTEGER—在原始來源資料集中的事故 ID。因不同資料集（A1、A2）的 ID 可能重疊，必須搭配 source_type 才能唯一定位一筆事故。
severity_idINTEGERFK外鍵，關聯到 accident_severity，標示此事故的嚴重程度（死亡 / 受傷 / 財損）。
district_idINTEGERFK外鍵，關聯到 district，標示事故發生所在的行政區，確保資料在空間範圍內的一致性。
longitudeDOUBLE PRECISION—事故發生地點的經度（WGS84），保留原始數值供計算使用。
latitudeDOUBLE PRECISION—事故發生地點的緯度（WGS84）。
geomgeometry(Point)—事故地點的空間點位，由經緯度建構而成，建有 GIST 索引。可用於「找出某路段附近 N 公尺內的所有事故」等空間查詢。
accident_edge_match 事故對應路段
將每筆事故「對應（snap）」到最近的路段邊，解決事故 GPS 座標與路網線條不完全重疊的問題。此表是事故資料與路段風險計算的橋接中間表。
屬性名稱資料型別鍵值詳細說明
match_idSERIALPK對應記錄的唯一識別碼。
accident_idINTEGERFK外鍵，關聯到 accident，指出是哪筆事故被對應。
edge_idINTEGERFK外鍵，關聯到 road_edge，指出事故被對應到哪條路段邊（最近鄰搜尋結果）。
distance_to_edge_mDOUBLE PRECISION—事故點到對應路段邊的距離（公尺）。距離越小代表對應越精準；可用於品質篩選，例如排除距離超過 50 公尺的疑似誤對應資料。
edge_risk_score 路段風險分數
彙整每條路段邊上的事故數量與嚴重度，計算綜合風險分數。此分數是路線規劃演算法中「避開危險路段」功能的核心數據來源。
⚙️ 計算公式 severity_score = A1事故數 × 3 + A2事故數 × 2
accident_density = 事故總數 ÷ MAX(路段長度km, 0.05km)
risk_score = severity_score ÷ MAX(路段長度km, 0.05km)

路段長度最小以 50 公尺計，避免極短路段的密度數值爆炸性放大。
屬性名稱資料型別鍵值詳細說明
edge_idINTEGERPK+FK同時為主鍵與外鍵，關聯到 road_edge。每條路段邊對應一筆風險記錄（1:1 關係）。
accident_countINTEGER—此路段上的事故總筆數（A1 + A2 合計）。
a1_countINTEGER—此路段上死亡事故（A1）的件數。
a2_countINTEGER—此路段上受傷事故（A2）的件數。
severity_scoreINTEGER—加權嚴重度分數。公式：A1×3 + A2×2，賦予死亡事故更高的懲罰權重，反映其社會成本。
accident_densityDOUBLE PRECISION—事故密度（件 / 公里）。避免長路段因絕對數量多而被高估，以密度標準化後可公平比較不同長度的路段。
risk_scoreDOUBLE PRECISION—最終風險分數，結合嚴重度與路段長度正規化後的結果。路線規劃演算法以此值作為「風險成本」，加入最短路徑計算中。
二
使用者 & 路線資料表
user_level 駕駛等級
定義系統中的使用者駕駛等級分類，決定路線規劃時風險的權重比例。新手會更積極地避開高風險路段。
屬性名稱資料型別鍵值詳細說明
user_level_idSERIALPK等級的唯一識別碼。
level_codeTEXTUK等級代碼，不可重複。BEGINNER（新手）、NORMAL（一般）、EXPERIENCED（熟練）。
level_nameTEXT—等級中文名稱，例如「新手駕駛」，用於前端 UI 顯示。
risk_weightINTEGER—風險權重（0–100）。數值越高代表路線規劃時越優先避開高風險路段。新手=80、一般=40、熟練=10。此值會被快照複製到 route_request.risk_weight 中。
min_scoreINTEGER—晉升至此等級所需的最低累積分數。新手=0（預設入門）、一般=500、熟練=2000。系統依 app_user.total_score 自動判斷等級。
descriptionTEXT—等級的文字說明，描述此等級駕駛的行為特徵與路線規劃偏好，用於 App 內說明頁面。
updated_atTIMESTAMP—此等級設定最後一次修改的時間戳記，供管理員追蹤設定變更歷程。
app_user 使用者帳號
儲存所有 App 使用者的帳號資料、個人資訊與等級狀態。
屬性名稱資料型別鍵值詳細說明
user_idSERIALPK使用者的系統唯一識別碼。
usernameTEXT—使用者在 App 中顯示的名稱（暱稱），允許重複，不作為登入識別。
emailTEXTUK電子郵件，作為帳號的唯一登入識別符，不可重複。
password_hashTEXT—使用者密碼的雜湊值（Hash）。資料庫不儲存明文密碼，保護帳號安全。應使用 bcrypt 或 Argon2 等演算法產生。
birth_dateDATE—出生日期，可用於年齡分布統計分析，或未來實作未成年用戶的功能限制。
license_dateDATE—駕照取得日期，可輔助系統判斷駕駛年資，或作為初始等級設定的參考依據。
addressTEXT—使用者居住地址（純文字記錄），非空間欄位，用於聯絡或統計用途。
home_latDECIMAL(10,8)—住家位置緯度，精確到小數點後 8 位（約 1mm 精度）。可作為路線規劃的預設起點。
home_lngDECIMAL(11,8)—住家位置經度，精確到小數點後 8 位。
user_level_idINTEGERFK外鍵，關聯到 user_level，代表使用者目前的駕駛等級。當 total_score 達到門檻時，此欄位應由應用程式邏輯更新。
total_scoreINTEGER—使用者累積的練習總分數（預設 0）。每次練習完成後依 score_earned 累加，達到門檻後觸發等級晉升。
roleTEXT—帳號角色（預設 user）。user 為一般使用者，admin 為管理員，用於後端 API 的權限控管（RBAC）。
created_atTIMESTAMP—帳號建立的時間戳記，自動記錄，用於稽核與新用戶趨勢分析。
user_route_preference 使用者路線偏好（弱實體）
弱實體（Weak Entity），每位使用者對應唯一一筆偏好設定，以 user_id 作為主鍵兼外鍵。記錄使用者對路線特性的個人偏好。
屬性名稱資料型別鍵值詳細說明
user_idINTEGERPK+FK同時為主鍵與外鍵，關聯到 app_user。確保每位使用者只有一筆偏好設定（1:1 關係）。
avoid_bridgeBOOLEAN—是否在路線規劃時避開橋梁。true 表示系統會排除含橋梁路段的路徑；預設 false。
avoid_tunnelBOOLEAN—是否在路線規劃時避開隧道。true 表示系統會排除含隧道路段的路徑；預設 false。
updated_atTIMESTAMP—偏好設定最後修改的時間戳記，記錄使用者何時調整設定。
route_request 路線請求
記錄使用者每一次的導航請求，包含起終點座標，以及當下的等級與偏好設定快照。快照設計確保歷史紀錄的資料一致性，不受使用者日後升級或修改偏好影響。
屬性名稱資料型別鍵值詳細說明
request_idSERIALPK路線請求的唯一識別碼。
user_idINTEGERFK外鍵，關聯到 app_user，記錄是哪位使用者發出此請求。
user_level_idINTEGERFK外鍵，記錄請求當下的駕駛等級快照。即使使用者日後升級，此筆歷史紀錄仍保留當時的等級，確保資料可重現性。
start_lngDOUBLE PRECISION—起點的經度座標。
start_latDOUBLE PRECISION—起點的緯度座標。
end_lngDOUBLE PRECISION—終點的經度座標。
end_latDOUBLE PRECISION—終點的緯度座標。
start_geomgeometry(Point)—起點的空間點位，由 start_lng/lat 建構。建有 GIST 索引，供「找最近路網節點」等空間查詢使用，是路線規劃演算法的輸入起點。
end_geomgeometry(Point)—終點的空間點位，用途同 start_geom，作為路線規劃的終點輸入。
risk_weightINTEGER—請求當下的風險權重快照（從 user_level.risk_weight 複製）。路線規劃演算法直接使用此值，確保即使等級參數日後調整，歷史路線仍可正確重算。
avoid_bridgeBOOLEAN—此次請求的避橋梁設定快照（從 user_route_preference 複製），同樣是歷史不可變設計。
avoid_tunnelBOOLEAN—此次請求的避隧道設定快照。
created_atTIMESTAMP—請求建立的時間戳記，可用於分析使用者的導航行為時段分布。
route 路線結果
儲存每次路線請求計算出的路線摘要資訊。一個請求可能產生多條路線（例如同時提供「最短距離」與「最低風險」兩條備選路線）。
屬性名稱資料型別鍵值詳細說明
route_idSERIALPK路線的唯一識別碼。
request_idINTEGERFK外鍵，關聯到 route_request，對應產生此路線的請求。
route_nameTEXT—路線名稱，可由系統自動命名（例如「最短路線」、「最安全路線」）或由使用者自訂，方便日後識別與收藏。
total_distance_mDOUBLE PRECISION—路線全程總距離（公尺），由各路段的 segment_distance_m 加總而來，顯示於導航 UI 上。
total_base_costDOUBLE PRECISION—路線的純距離成本總和（不含風險加權），反映若只考慮最短距離時的基礎行駛成本。
total_risk_scoreDOUBLE PRECISION—路線的總風險分數，由各路段的 segment_risk_score 加總而來，用於向使用者展示「這條路線整體有多危險」。
total_final_costDOUBLE PRECISION—路線最終綜合成本，結合距離成本與風險分數（依 risk_weight 加權）的總和，是多條備選路線排名比較的依據。
created_atTIMESTAMP—路線計算完成的時間戳記。
route_segment 路線路段明細（弱實體）
弱實體，記錄路線中每一段路段邊的詳細成本與順序，是路線的逐段分解明細表。主鍵為 (route_id, sequence_order) 複合鍵。
屬性名稱資料型別鍵值詳細說明
route_idINTEGERPK+FK複合主鍵之一，外鍵關聯到 route，標示此路段屬於哪條路線。
sequence_orderINTEGERPK複合主鍵之一，此路段在整條路線中的行駛順序編號（1, 2, 3...）。依此欄位排序可還原完整的行駛路徑。
edge_idINTEGERFK外鍵，關聯到 road_edge，指出此順序的路段對應到路網中的哪條邊。
segment_distance_mDOUBLE PRECISION—此路段的長度（公尺），來自 road_edge.length。
segment_base_costDOUBLE PRECISION—此路段的純距離成本（不含風險），來自 road_edge.cost。
segment_risk_scoreDOUBLE PRECISION—此路段的風險分數，來自 edge_risk_score.risk_score，代表此段路的危險程度。
segment_final_costDOUBLE PRECISION—此路段的最終綜合成本（距離成本 + 風險分數 × risk_weight）。演算法選路時以最小化各路段 segment_final_cost 的加總為目標。
user_practice_history 使用者練習歷史
記錄使用者每次駕駛練習的歷程、狀態與得分。累積分數達到門檻後可觸發等級晉升，是整個成長系統的核心記錄。
屬性名稱資料型別鍵值詳細說明
practice_idSERIALPK練習記錄的唯一識別碼。
user_idINTEGERFK外鍵，關聯到 app_user，記錄是哪位使用者的練習記錄。
route_idINTEGERFK外鍵，關聯到 route，記錄此次練習使用的路線。允許 NULL，代表非路線導向的自由練習模式。
practice_timeTIMESTAMP—練習開始的時間戳記（預設為當下時間）。可用於統計使用者的練習頻率與高峰時段。
statusTEXT—練習狀態（預設 in_progress）。可能的值：in_progress（進行中）、completed（成功完成）、abandoned（中途放棄）。只有 completed 狀態才應計分。
selected_difficultyTEXT—使用者此次選擇的練習難度，例如 easy、medium、hard。難度越高，score_earned 的乘數越大，鼓勵使用者挑戰更難的路線。
score_earnedINTEGER—此次練習獲得的分數（預設 0）。完成後累加至 app_user.total_score，達到 user_level.min_score 門檻時觸發晉升邏輯。
